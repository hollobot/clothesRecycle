# clothesRecycle

校园旧衣回收与流转平台（Monorepo）。
本 README 用于明确系统功能边界、用户权限与核心业务流程。

## 1. 系统目标

- 为普通用户提供旧衣发布、浏览、认领、积分兑换、回收点投递与消息通知能力。
- 为管理端提供物品审核、回收点管理、用户管理与数据统计能力。
- 通过角色权限控制（超级管理员 / 校区管理员）保证跨校区数据隔离。

## 2. 系统边界（用于 UML 系统边界框）

**「校园旧衣回收与流转平台」**

## 3. 角色与权限边界

### 3.1 普通用户

- 可注册/登录、维护个人资料。
- 可发布物品、查看物品、收藏、下单、取消订单、完成订单。
- 可查看积分、签到、积分兑换礼品、查看兑换记录。
- 可查看回收点、排行榜、消息通知。

### 3.2 校区管理员（普通管理员）

- 仅可操作**所属校区**数据。
- 物品审核：仅本校区物品（审核、强制下架）。
- 回收点管理：仅本校区回收点（查询/新增/编辑/状态变更）。
- 用户管理：仅本校区用户（查询/详情/新增/编辑/状态变更/删除）。
- 数据看板：仅展示本校区统计数据。
- 礼品管理：**无权限**。

### 3.3 超级管理员

- 可跨校区管理与查看。
- 具备所有校区管理员权限。
- 额外具备：校区管理、管理员账号管理、礼品管理（含兑换核销）。

## 5. 核心流程（可直接转活动图/时序图）

### 5.1 普通用户：发布并流转物品

1. 用户登录。
2. 填写物品信息并上传图片。
3. 提交发布后，物品状态进入待审核。
4. 管理员审核通过后，物品上架可见。
5. 其他用户下单认领，进入订单流程。

### 5.2 普通用户：积分兑换礼品

1. 用户进入礼品商城查看礼品（含图片、积分、库存）。
2. 发起兑换请求并二次确认。
3. 系统扣减积分、减少库存，生成兑换码。
4. 用户在“我的兑换记录”查看状态。

### 5.3 校区管理员：物品审核流程

1. 管理员登录后进入本校区待审核列表。
2. 审核通过/驳回并填写说明。
3. 系统更新物品状态并记录审核人。
4. 若越权操作他校区数据，系统拒绝并返回权限错误。

### 5.4 超级管理员：礼品管理流程

1. 进入礼品管理页查看礼品列表（含图片）。
2. 新增/编辑礼品时可上传图片并预览。
3. 提交前进行对话框确认。
4. 上下架、核销等写操作均需二次确认。

## 7. 模块划分

### 7.1 客户端模块

- 认证模块（登录/注册）
- 个人中心模块（资料、积分、消息）
- 物品模块（发布、列表、详情、我的发布）
- 订单模块（认领、取消、完成）
- 礼品模块（商城、兑换、兑换记录）
- 回收点与排行榜模块

### 7.2 管理端模块

- 数据看板
- 物品审核管理
- 回收点管理
- 用户管理
- 礼品管理（仅超级管理员）
- 校区管理（仅超级管理员）
- 管理员账号管理（仅超级管理员）

### 7.3 后端服务分层

- Controller：接口入口、参数校验、统一返回结构。
- Service：业务规则与权限控制。
- Mapper：数据库访问。
- Model：DTO/PO/VO。
- Security：Sa-Token 登录态、角色与校区权限约束。

## 9. 非功能需求（可在用例图外文字说明）

- 权限安全：校区数据隔离，禁止校区管理员跨校区操作。
- 一致性：库存与积分扣减具备业务校验。
- 可用性：关键写操作提供二次确认。
- 可维护性：前后端分层清晰，接口统一返回 `Result<T>`。
