## 功能一：物品发布与审核

**核心说明**：用户发布旧衣，经管理员审核后上架展示，支持主动 / 强制下架。

**简化流程**：

1. 用户填写物品信息提交 → 进入**待审核**状态
2. 管理员审核：
   - 通过 → 物品**已上架**，广场公开浏览
   - 不通过 → 物品**已驳回**，用户可修改后重新提交
3. 下架操作：用户主动下架 / 管理员强制下架 → 物品**已取消**

------

## 功能二：订单认领流转

**核心说明**：买家发起认领，卖家确认后线下交接，完成交易并结算积分。

**简化流程**：

1. 买家选择物品提交认领 → 系统校验物品状态

   - 已被他人锁定 → 认领失败
   - 可用 → 创建**待确认**订单，积分支付则冻结对应积分

   

2. 卖家确认交付 → 订单**已确认**，双方约定线下交接

3. 买家确认收货 → 订单**已完成**，自动结算积分

4. 异常处理：待确认阶段取消 / 超时未操作 → 订单作废，释放积分和物品资源

------

## 功能三：积分获取与礼品兑换

**核心说明**：用户通过指定行为积累积分，可用于兑换平台礼品或认领积分物品。

**简化流程**：

- **积分获取**：每日签到得积分；免费送出的物品交易完成，自动发放奖励积分

- 礼品兑换

  1. 用户选择礼品提交兑换 → 校验积分和库存
     - 积分不足 / 礼品无库存 → 兑换失败
     - 校验通过 → 扣减积分，生成**待核销**兑换码
  2. 线下网点核销兑换码 → 兑换完成；超期未核销 → 积分自动退回

## 功能四：管理端权限与后台管理

**核心说明**：两级管理员（超级 / 校区）分别管理全局和对应校区资源，操作受权限限制。

**简化流程**：

1. 管理员登录 → 验证账号密码
   - 验证失败 → 提示错误信息
   - 验证成功 → 按权限分配数据范围（超级管理员管全局，校区管理员仅管绑定校区）
2. 核心操作：用户管理、物品审核、礼品管理、校区 / 回收点管理（所有操作自动记录日志）
3. 权限越界：校区管理员尝试操作其他校区数据 → 系统直接拒绝
